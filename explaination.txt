| Testing Type            | What It Checks                       | Example from Your Project                |
| ----------------------- | ------------------------------------ | ---------------------------------------- |
| **Unit Testing**        | One small part (like a function)     | Test only the `predict()` function       |
| **Integration Testing** | Connection between parts             | Form → Prediction → Display              |
| **System Testing**      | Entire app functionality             | Login, predict, view logs                |
| **Acceptance Testing**  | Final check if it meets expectations | Reviewer's or guide’s demo test          |
| **Regression Testing**  | After changes, test old features     | Adding a feature → Rechecking everything |

white box testing - the testing that we can see code and tests the code or functions and parameters
black box testing - the testing are done without knowing the actual code i.e., we can test the outputs by inputs or interactions or integrations or verifying logins and signins
